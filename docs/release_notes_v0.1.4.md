# ClipScope v0.1.4 リリースノート
リリース日: 2026-03-24

## 変更概要
Twitch 認証の継続利用を改善し、長時間運用時にトークン期限切れで監視が止まりにくいようにしました。

## 主な改善

- リフレッシュトークンを使った自動再認証を追加
- `Get Clips` 実行前にアクセストークン有効期限を確認し、必要時は自動更新
- 対象は手動更新時と監視ポーリング時の両方
- Twitch API 呼び出しで `401 Unauthorized` が発生した場合、再認証後に 1 回だけ再試行
- 起動時にも、保存済みリフレッシュトークンがあれば自動再認証を試行
- 新しいアクセストークン、有効期限、リフレッシュトークンを SQLite に保存するよう改善

## 互換性・注意点

- 既存 DB（`%APPDATA%\ClipScope\clipscope.db`）はそのまま利用できます
- リフレッシュトークン自体が失効している場合は、再度 Twitch 認証が必要です
- Windows native モードでまれに発生する低頻度フリーズ問題は、今回のリリースでは既知事項のままです
